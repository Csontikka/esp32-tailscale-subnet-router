FIREWALL PAGE
=============

CONCEPT
───────
4 ACL lists shown as 4 collapsible cards, stacked vertically.
Each card has: header with list name + direction + hit summary + [Add Rule] button.
Body: rule table (scrollable on mobile).
Below all 4 cards: an "Add Rule" form card (or inline row on desktop).

MOBILE (≤639px)
───────────────

Each ACL card stacks vertically.
Rule table uses .table-wrap overflow-x:auto — columns scrollable.
On very narrow screens the table shows condensed columns:
  Proto | Src→Dst | Action | Hits | Del
  (Src/Dst IP+port merged into one cell with line break)

Add Rule form collapses to single-column layout.
[+ Add Rule] button is full-width per card header.

ACTION BADGE COLORS:
  Allow         → .badge-success     (green)
  Allow+Monitor → .badge-success + small eye icon overlay
  Deny          → .badge-danger      (red)
  Deny+Monitor  → .badge-danger  + small eye icon overlay

CLEAR LIST:
  Each card header has a secondary [...] overflow menu (3-dot icon).
  Clicking opens: "Clear list" option → confirm modal before proceeding.
  This avoids accidental clears vs the always-visible [+ Add] button.

HIT COUNT:
  Rendered as plain text number. Format with locale thousands separator.
  Resets to 0 on device reboot (NVS saves rules, not hit counts).
  No last-hour windowing in v1 (add as future enhancement).
