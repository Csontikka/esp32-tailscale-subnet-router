SYSTEM PAGE
===========

LAYOUT CONCEPT
──────────────
Single-column, card-per-section layout on all breakpoints.
Sections ordered by risk: safe config changes first, destructive actions last.
Danger zone card always at the bottom, separated by extra spacing.

DANGER ZONE INTERACTION:
  All 3 actions → showConfirm() modal before any request is sent.
  Modal title = action name, body = consequence text, [Cancel] / [Confirm] buttons.
  Confirm button is .btn-danger (red).
  After confirm: spinner shown in place of button, POST to /api/factory-reset etc.
  On success: toast notification + (for factory-reset) redirect countdown.

OTA UPLOAD FLOW:
  1. User selects .bin file → filename shown in label span.
  2. [Upload Firmware] → XHR POST /api/ota-upload with progress bar.
  3. Progress bar fills to 100%.
  4. On success: card body replaced by reboot countdown + redirect to /.
  5. On error: inline error message below bar, bar resets.
  No page navigation during upload (warn user if they try to leave).

FILE INPUT STYLING:
  Native <input type="file"> hidden.
  Styled <label> wraps it: .bg-input border border-dashed border-border.
  Hover state: border-accent, background: accent-muted.
  Selected filename shown in <span> inside label.

MOBILE ADJUSTMENTS:
  All grid-cols-2 form groups collapse to 1 column.
  Danger zone buttons are full-width.
  Password fields: show/hide toggle essential for mobile usability.
