Layout: desktop uses grid-cols-2 — config form on left, live peers table on right.
The peer table is NOT inside the form; it's a separate card driven by /api/status
polling (no save button, read-only, refreshes with the 30s interval).

PEER TABLE COLUMNS (desktop, .table inside .card):
  Online dot | Hostname | Tailnet IP | Path | RTT | Last seen | Role badge

DERP REGION SELECT:
  ~30 options grouped by continent.
  "Auto (closest)" is the default selected option.
  Custom server URL option at bottom: user types in a text field
  that appears when "Custom..." is selected.

MTU INTERACTION:
  Fixed radio selected → number input enables (was disabled+greyed).
  Auto radio selected  → number input disables+greys, shows "(1420 direct / 1280 DERP)".
  MTU change does NOT require reboot — it is applied live via /api/mtu POST.
  The sticky footer text changes to "Applied immediately" when only MTU changed.
